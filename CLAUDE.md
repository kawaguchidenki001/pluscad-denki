# 電気図 配置・見積アプリ（plusCAD電気α 型）

## プロジェクトの目的
PDF/DXF/JWWの図面を下敷きに、電気設備シンボルを配置して図面を仕上げつつ、
配置数と配線長から見積を自動作成するブラウザ完結PWA（GitHub Pages + GAS + Drive、deploy.zip一括更新）。
plusCAD電気αと同じ「配置ベース」（白紙/下敷きにシンボルを置く→拾う）方式。

**このドキュメントの確定事項は再検証不要。新しい変更時はまずこのファイルと既存ソースを照合してから進めること。**

## 作業原則（厳守）
- 速さより確実性。憶測で進めず、実ファイル・一次資料で検証してから結論を出す。
- 記号形状・寸法・座標は、必ず標準図PDFの該当ページを実際に開いて突合してから確定する（テキストの名称・傍記だけで形を断定しない）。
- 複数方針がありうる場合は選択肢を提示してKの判断を待つ（タップ式）。
- 完了報告は簡潔に。コード全文をチャットに貼らず、成果物はファイル/zipで渡す。

## モデル指針
- シンボル整備・自動均等配置・自動配線・PDF下敷き・見積UI＝Opus領域（Fableの固有強みは不要だがFable maxで回しても品質は問題ない）。
- **完成図の DXF/JWW 書き出し（既存バイナリエンジン拡張）＝Fable が活きる工程。**

## 実装ロードマップ（配置ベース）

### Step A：下敷き拡張（PDF/画像は実装済み。DXF/JWW追加）
- PDF：PDF.jsで1ページ目を画像化。ベクターPDFは将来パースも検討（ベクターに限りCAD変換可）。
- DXF/JWW：既存パーサ(dxf_parser.js / jww_reader.js)でSVG描画。DXFは実座標なので較正不要。
- スケール較正：2点クリック＋実寸mm入力（PDF/画像用）。実装済み。

### Step B：照明の自動均等配置 ★次の主機能
配灯の実務ルール（Panasonic P.L.A.M / 岩崎電気 / JIS Z 9110 準拠）：
- 全般照明は取付間隔Sを等間隔、**壁際の器具と壁の間隔＝S/2 以下**（壁ぎわをよく使う場所は1/3）。
- 実装＝部屋を矩形（または多角形）指定 → 台数 nx×ny 指定 → 各方向を台数で等分割し**各セル中央**に配置：
  - `x = x0 + (W/nx)*(i+0.5)`, `y = y0 + (H/ny)*(j+0.5)`（i=0..nx-1, j=0..ny-1）
  - これで自動的に「壁際=ピッチ1/2、器具間=ピッチ」になる。1台指定なら部屋中央。
- 台数の目安算出（任意）＝光束法：`N = E×A/(F×U×M)`、室指数 `K = X×Y/((X+Y)×H)`。
  - E:設計照度[lx]、A:室面積[m²]、F:器具光束[lm]、U:照明率、M:保守率、H:器具～作業面高さ。
  - 計算照度は平均照度以上・目安130%以内。器具ピッチ 2.5〜3.3m、均整度0.5以上。
- 較正済み(mmPerPx)なら、部屋寸法・ピッチを実寸[m]で扱える。

### Step C：自動配線 ★
屋内配線図は**単線図**（1本線）。電源→スイッチ→照明、電源→コンセントを結ぶ。実務では同一回路を示すのが主目的で、スイッチのジョイントボックス結線は省略して見やすく描く。
- 実装＝各シンボルに「回路番号」を割当 → 同一回路のシンボルを**最近傍で数珠つなぎ（＝実務の"送り"配線、MST/最近傍）**＋スイッチ→対応照明を結線。
- 線種：天井隠蔽配線＝実線が基本。較正済みなら配線長を集計→見積へ。

### Step D：見積連携
- 配置シンボル（種別×個数）＋配線長 → 品目マッピング表 → 岐阜県単価マスタ照合 → 見積。
- 既存の見積アプリ(電気図見積アプリ_JWW対応_index.html)の単価連携・見積出力ロジックを流用。

### Step E：完成図の書き出し ★Fableが活きる工程
- 印刷用PDF：下敷き＋配置シンボルを合成。
- DXF/JWW出力：配置シンボルをDXFエンティティ化 → 既存 dxf_jww_webapp のエンジンでJWW/DXF書き出し（plusCADにない強み）。
- **既存バイナリエンジン(schema 700, CString BYTE-first, WriteCount WORD-first, little-endian)は再検証不要の確定仕様。実ファイルとのバイト突合で検証すること。**

## MVPの現状データ構造（HTML内）
- `placed[]` … 配置シンボル `{uid,type,x,y,rot}`（x,yはworld座標px）
- `SYMBOLS[]` … 記号定義 `{id,nm,cat,color,g(SVG本体)}`
- `mmPerPx` … 較正係数、`scale/tx/ty` … ビュー変換
- 保存＝配置＋下敷きdataURL＋較正のJSON。集計＝種別ごとの個数。
