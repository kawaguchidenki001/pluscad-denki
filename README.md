# pluscad-denki

plusCAD電気α型の電気図配置・見積アプリ。ブラウザ完結PWA。

## 現状
- MVP: PDF/画像を下敷きに、標準図準拠の電気シンボル60種をタップ配置。種別集計、保存/読込、スケール較正まで動作。
- Step B（照明の自動均等配置）実装済み: 「自動配灯」で部屋を対角2点指定 → 台数 nx×ny を指定（または光束法 N=E×A/(F×U×M) で自動算出）→ 各セル中央に均等配置（壁際＝ピッチ1/2）。較正済みなら寸法・ピッチ・壁際を実寸[m]で表示し達成率も判定。配灯範囲は `rooms[]` として保存。
- Step C（自動配線・単線図）実装済み: 「自動配線」で部屋・近接からシンボルを回路へ自動グループ化し、同一回路を最近傍で数珠つなぎ（MST）＋スイッチ→最寄り照明＋分電盤→各回路の電源送りを自動結線。回路ごとに色分け、較正済みなら配線長を回路別・合計で集計（CSVにも出力）。「手動配線」で2点タップ結線／線タップ削除の手修正も可能。配線は `wires[]` として保存。
- Step D（見積連携）実装済み: 「見積」で配置数（種別×個数）＋配線長を品目に自動マッピングし、岐阜県単価マスタ（`unit_prices.json` 9,154件・複合単価＋材料/労務/経費内訳）と照合して見積明細を自動生成。分類→品目→規格の3段選択で仕様を確定でき、数量編集・除外・信頼度表示（黄=自動/緑=確定/赤=未マッチ）に対応。現場管理費/一般管理費/値引/消費税/配線余長を調整し、材料費計・労務費計・経費計・直接工事費・見積合計を即時計算。見積CSV（Excel）出力。単価マスタは同梱ファイル→サーバ→ローカル選択の順で読込（オフライン可）。見積設定は `estimate` として保存。
- Step E（完成図の書き出し）実装済み: 「書き出し」メニューから **完成図DXF** / **完成図JWW** / **印刷（PDF保存）** を出力。シンボルSVG定義を実行時にCADエンティティ（線・円・円弧・文字・塗りソリッド）へ厳密変換し、DXF（Shift-JISテキスト）とJWW（`lib/dxf_jww/` の実証済みバイナリエンジン・schema 700）を同一の中間表現から生成。座標は較正済み実寸[mm]、JWWは自動縮尺（A3基準）・カテゴリ別レイヤ・回路別線色。印刷は下敷き＋配線＋シンボルを合成してブラウザの印刷（PDF保存）へ。
- 保存管理（ブラウザ内保存）実装済み: 「保存」でこの端末のブラウザ内（IndexedDB）に図面を保存。トップページの一覧、およびヘッダの「読込」で開いた一覧から過去の保存をタップで開ける（開く／削除）。バックアップ・別端末への移動用に「ファイルに書き出し（.json）」／「保存ファイルから開く」も用意。別の下敷きを開くと保存先はリセットされ、既存図面を無断で上書きしない。ヘッダとトップにアプリのバージョン（例 `v0.6`）を表示し、更新を確認できる。
- 操作性の改善（v0.7）: タッチ端末でヘッダのズーム（＋／－／全体）・選択バーのボタン（削除・複製・回転・コピー）が反応しない不具合を修正（stageがタップを横取りしていた根本原因を解消）。**戻る/進む（Undo/Redo）**をヘッダに追加（配置・削除・移動・回転・複製・コピー・配線・配灯・色・全消去に対応、Ctrl+Z / Ctrl+Y も可）。選択シンボルの**複製**ボタンを追加。選択バーを狭い画面で折返し表示にして全ボタンをタップ可能に。「配線編集」を**手動配線**に改称し説明を明確化。
- 操作性パック（v0.8）: **範囲選択**（矩形ドラッグで複数選択→一括移動・削除・複製・方向コピー・横/縦揃え・横/縦等間隔）、**文字・注記ツール**（タップ配置、編集・サイズ・回転・色・移動。印刷/DXF/JWWにも出力）、**グリッド・スナップ**（間隔可変[較正時mm]、配置・移動・整列が格子に吸着、格子表示つき）、**記号パレットの検索・★お気に入り（長押し登録）・最近使った記号**。Delete/Escキー対応。
- アプリ基盤パック（v0.9）: **PWA化**（`manifest.webmanifest`＋`sw.js`。ホーム画面に追加してアプリのように起動でき、一度開けばオフラインでも動作。アプリシェル＝index.html/lib/単価マスタ/アイコンをキャッシュ）。**自動保存**（編集のたびにブラウザ内へ自動バックアップ。うっかり閉じても、次回トップページの「前回の続きから」で復元）。
- 図面パック（v0.10）: **DXF/JWWを下敷きに**（既存パーサでベクター描画。実mm座標で読み込むため**較正不要**、下敷き濃さ調整も可）。**図枠・タイトル欄・凡例表の自動生成**（印刷時に図枠＋タイトル欄〔工事名/宛先/見積番号/作成日/縮尺〕＋凡例表〔使用記号一覧＋個数〕を付加。ON/OFF可）。
- 複数ページPDF対応（v0.11）: 複数ページのPDFを開くと**サムネイル一覧からページを選択**して下敷きにできる。ヘッダの「ページ n/N」ボタンでいつでも別ページに**切替**可能（現状は選んだ1ページを下敷きにする方式。ページごとに別々の配置を持つ「シート」管理は今後の拡張）。
- 配線の線種・条数（v0.12）: 配線集計パネルで**回路ごとにケーブル（VVF1.6-2C／2.0-3C 等）を選択**でき、見積は**線種別に配線長を集計**して単価マスタと照合。「線種を図に表示（傍記）」で各配線に線種ラベルを表示し、印刷にも反映。
- Excel見積出力（v0.13）: 見積を**罫線・書式付きの.xlsx**で書き出し（見積書タイトル・工事名等ヘッダ・明細・合計）。依存ライブラリなしでZIP＋OOXMLを内製生成するため**オフラインでも動作**。従来のCSV出力も併存。
- 背景反転（v0.14）: 下敷きの濃さ調整とは別に、ヘッダの「背景反転」で**白い図面を黒背景に反転**（暗所や液晶での視認性向上）。配置シンボルは反転しない。反転状態は保存・自動保存に含む。
- DXF堅牢化（v0.15）: DXF下敷きの取り込みで **ELLIPSE（楕円・楕円弧）／SPLINE（近似表示）／DIMENSION（寸法図形）** に対応。**バイナリ形式のDXF**を検出して「ASCII DXFで保存し直す」旨を案内。JWW取り込みが未対応要素で途中打ち切りになった場合は「部分的に取り込みました」と正直に表示。
- レイヤー分け（v0.16）: JW-CAD相当の**レイヤー機能**。配置シンボルは種別（照明/コンセント/スイッチ/盤・機器/弱電・防災/配線・接続材）で自動的にレイヤーへ振り分け、配線・文字・下敷きも各レイヤーに。右パネルの一覧で**表示ON/OFF・ロック（表示のみ・編集不可）・この層だけ表示・レイヤー追加/改名/削除**が可能。選択した記号は「層」セレクトで別レイヤーへ移動（範囲選択なら一括移動）。**非表示レイヤーは印刷・DXF/JWW書き出しからも除外**（画面と一致）。レイヤー構成は保存・復元。
- 読取点スナップ（v0.17）: JW-CAD相当の**読取点吸着**。ヘッダの「読取点」をONにすると、配置・移動・文字配置のとき、**DXF/JWW下敷きの線の端点・角・ポリライン頂点・円/円弧の中心や端点**、および**配置済み記号の中心**に吸着（吸着位置に⊕マーカー表示）。下敷きトレースが正確に。吸着半径は表示倍率に追従。グリッド吸着と併用時は読取点を優先。ラスタ下敷き（PDF/画像）は幾何情報がないため記号中心のみ対象。ON/OFFは保存・復元。
- 配線の線種（v0.18）: 配線集計パネルで**回路ごとに施工方法の線種**を選択（実線＝天井隠蔽／破線＝床隠蔽／点線＝露出／一点鎖線＝地中ほか）。**画面・印刷・DXF出力**に反映（DXFはR12 LTYPE表＋線分linetype）。JWWは現状すべて実線（penStyleの線種番号を実ファイルで確認後に対応予定）。設定は保存・復元。
- 印刷の縮尺指定（v0.19）: 書き出しメニューに**用紙サイズ（A4/A3×縦横）と縮尺（自動フィット／1:20〜1:500）**を追加。縮尺指定時は較正値から実寸を算出し、用紙に対して**1:Sの物理サイズ**で配置（`@page`で用紙寸法を指定）→ブラウザ印刷100%で正確な縮尺の図面に。用紙に収まらない場合は警告。縮尺指定には「縮尺を設定」（またはDXF/JWW下敷き）が必要で、未設定なら自動フィット。
- 印刷の縮尺表示・較正の堅牢化（v0.20）: 固定縮尺で印刷するとタイトル欄の縮尺欄に **「1:100」等** を表示（自動フィット時は「実寸(mm)」）。較正で2点が近すぎる場合を弾き、`mmPerPx=∞` を防止。
- 自由線（作図線）ツール（v0.21）: ヘッダの「作図線」で**連続線（ポリライン）**を作図（頂点をタップ→「確定」、「1つ戻す」「✕」、Enter確定/Esc取消）。読取点・グリッド吸着つき、頂点自体も読取点になる。専用「作図線」レイヤーで表示/ロック、選択→削除、印刷・DXF/JWW（DRAWレイヤー）出力、保存/復元/Undo対応。間仕切り・敷地境界・補助線・別ルートの下書き等に。※複線/トリムは次段。
- 複線・選択修正（v0.22）: 作図線に**複線（平行複写）**を追加（間隔を入れ「複線」/「逆」で左右にオフセット。ポリラインは各線分をオフセットし交点でミター接合）。あわせて、記号/文字を選んだりツールを切替えたときに**作図線の選択バーが残る不具合を修正**。
- 複線のミター制限（v0.23）: 複線で鋭角/浅角の折れ点がある場合にミター点が遠方へ暴走するのを抑制（マイターリミット≒間隔×4を超えたら法線オフセット点で代用）。
- 全ステップ（A〜E）＋保存管理・操作性改善・PWA/自動保存・DXF/JWW下敷き・図枠・複数ページPDF・配線線種・Excel見積・レイヤー分け・読取点スナップ・配線線種(施工)・印刷の縮尺指定・自由線作図・複線 実装完了。詳細は `CLAUDE.md` を参照。

## ローカル起動
`index.html` を Chrome / Edge / Firefox で開く。PDF.js は CDN から読み込むためネット接続下で。

## GitHub Pages で公開する
1. このフォルダを GitHub リポジトリに push
2. Settings → Pages → Deploy from a branch → main / root
3. `https://<user>.github.io/<repo>/` でアクセス

## クラウド版 Claude Code で続きを実装する
1. GitHub にリポジトリを push（Windowsのブラウザだけで完結）
2. Windows のブラウザで `https://claude.ai/code` を開く
3. GitHub を接続し、このリポジトリでクラウドセッションを起動
4. 最初のプロンプト:「CLAUDE.md を読んで現状を把握して、Step B（照明の自動均等配置）から始めて」

Windows 側にインストールは不要。実行は Anthropic 側のクラウドVM。

## フォルダ構成
- `index.html` … アプリ本体
- `unit_prices.json` … 岐阜県単価マスタ（9,154件）。`tanka-search` の同名ファイルの同梱コピー。見積（Step D）がオフラインでも動くよう相対パスで読み込む。
- `CLAUDE.md` … 設計仕様書（クラウドClaude Codeが毎回自動で読む）
- `symbols/` … 標準図準拠シンボル60種
  - `svg_mono/` 個別SVG（黒・実図面向け）
  - `svg_color/` 個別SVG（分類色付き）
  - `symbols.js` / `symbols.json` 定義データ
- `lib/dxf_jww/` … DXF⇔JWW変換エンジン（完成図書き出し用、Step E で使用）
- `reference/` … 国交省標準図PDF、図記号抽出テキスト
